Guía de uso — WCG One

Referencia rápida para demo interna y uso gerencial.

¿Qué es cada módulo?
CRM — Clientes
Maestro único de entidades, contactos, productos relacionados, interacciones y tareas de seguimiento comercial.
Balón de Riesgo
Cuadro operativo de mora y saldos vencidos por operación, basado en snapshots históricos (leasing y similares).
PGO — Operación
Indicadores de helpdesk: tickets, tiempos de atención, cumplimiento de SLA y resúmenes por período.
PGC
Plan de gestión comercial operativo integrado en este mismo proyecto Django (/pgc/, /tablero/).
Ruta sugerida para demo (15–20 min)
  1. 1. Panel principal — KPIs globales e importaciones recientes.
  2. 2. CRM — Clientes — listado, resumen, detalle, export CSV.
  3. 3. Riesgo — Comando Balón — mora, saldos vencidos, historial por operación.
  4. 4. PGO — Dashboard — tickets abiertos/cerrados y SLA.
  5. 5. Admin Django — soporte operativo y revisión de lotes de importación.
Sembrar datos demo
python manage.py seed_wcg_demo
python manage.py seed_wcg_demo --fresh

El comando crea entidades, contactos, snapshots, tickets y lotes de importación simulados marcados como origen=demo.
